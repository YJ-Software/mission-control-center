# 🌅 晨報分段輸出格式（統一版）

> 本段定義所有晨報 cron 的一致輸出格式與品質規範。
> 下方有本次任務的具體指示，請一併遵守。

---

## ⚠️ 日期與路徑規則（最高優先，違反即失敗）

### 日期（已預填，**無需執行 date 命令**）
- **今日日期**：`${DATE_HYPHEN}`
- **檔名日期**：`${TODAY}`（用於所有檔名）
- ⛔ **嚴禁**猜測、推算、或另行執行 `date` 命令取得日期。直接使用上方預填值。

### 路徑（固定，不可修改）
所有晨報暫存檔的**唯一正確路徑**是：
```
${TMP_DIR}/
```
⛔ 不是任何其他路徑。
寫入前請確認路徑開頭是 `${TMP_DIR}/`。

---

## A) 全域品質要求（必須遵守）
- **${LANGUAGE}**。
- 每則資訊必須包含：
  1) **一句話重點（結論先講）**
  2) **2 句補充**（背景/關鍵數字/影響）
  3) **來源連結**（找不到就換來源/換一則）
- 絕不只貼標題或只貼連結。
- **📅 時效性強制要求（新聞類區塊）**：
  - 所有新聞必須是**當日（${DATE_HYPHEN}）或前一日**發布
  - 搜尋時使用 freshness 參數：`freshness: "pd"` (past day)
  - 若搜尋結果包含舊新聞（超過 48 小時），**立刻跳過，尋找下一則**
  - 寧可少一則，也不要放過時新聞
- **🔗 URL 去重規則（強制）**：
  - ⛔ **同一段落內禁止出現重複 URL**。每個 URL 只能引用一次。
  - ⛔ 若本 prompt 末尾附有「前日已使用的 URL」清單，**嚴禁再次引用清單中的任何 URL**。遇到相同 URL 請跳過，尋找其他來源。

---

## B) 全域工具優先序（除非額外指定，否則必須遵守）
### 搜尋
1. **第一優先 web_search (請每秒只發送一次請求)**
   - **必須加上時效參數**：`freshness: "pd"` (past day) 或 `freshness: "pw"` (past week)
   - 新聞類搜尋一律用 `"pd"`
2. **備援：web-search-prime（mcporter）**（只在 web_search 不可用時）
   - 同樣加上：`search_recency_filter: "oneDay"`

### 網頁讀取
1. **web-reader（mcporter）優先**
2. **備援：web_fetch**

### 容錯（最重要）
- 429 / rate limit：**立刻停止 web_search** → 改 web-search-prime
- 403 / Cloudflare / Robot check：**不要卡住重試** → 直接換來源/換網站
- web_search 與 web-search-prime 都無法使用時
  - 直接使用 OpenClaw Browser Relay 開啟網站
  - 使用 agent-browser MCP 開啟網站
- **搜尋結果中發現舊新聞（發布時間超過 48 小時）**：立刻跳過該則，繼續找下一則

---

## C) 統一輸出骨架（每個 cron 都照這個）

### 1) 段落標頭
用這一行當作訊息第一行：
- `【晨報 ${TOPIC_INDEX}/${TOPIC_TOTAL}｜HH:MM｜${TOPIC_NAME}】`

### 2) 內容區塊
每個晨報「區塊」用二級標題：
- `## <emoji> <區塊標題>`

### 3) 每則條目的固定格式（強制）
每一則使用同一個 bullet 模板：
- `- **<一句話重點>**：<補充句 1> <補充句 2>[來源](URL)`

### 4) 可選：小節（若該區塊需要分組）
用三級標題：
- `### 小節名稱`

---

## D) 段末結尾
- 除了最後一段（${TOPIC_TOTAL}/${TOPIC_TOTAL}）以外，其它段落不用固定結尾。
- 最後一段段末加：`✅ 今日晨報 ${TOPIC_TOTAL} 段已全部送出`
